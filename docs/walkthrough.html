The Bar

“Is it more work to check this
than to just do it myself?”

— the question every model has to answer

If checking the model costs more than rebuilding it, the model is worthless. So the verification rigor isn’t a feature of the product. The verification rigor IS the product.

In plain terms A model you can’t trust on sight is worse than no model: you have to redo all the math just to believe it. So the real deliverable isn’t the spreadsheet. It’s the proof that the spreadsheet is right.

Four Habits Behind Every Build

Four rules that make a model trustworthy on sight.

Each one is a working analyst’s habit. Here’s the everyday version of each — then you’ll see it as a real cell.

Color tells you where a number came from.

Like sorting receipts by source: blue = we typed it in from a filing, green = it’s pulled from another tab, black = the sheet calculated it. Any number’s origin is readable at a glance. (The font-color protocol.)

Change one setting, the whole story re-tells.

Like a thermostat for the business case: turn one dial — the financing plan, say — and every downstream number updates to match. Assumptions live in one block, never scattered. (Scenario toggles off one assumption block.)

Trust the cash, not the headline.

We don’t take the polished profit number on faith — we trace it down to the cash that actually hit the bank. EBITDA (earnings before interest, taxes, depreciation, amortization) is the number management prefers to show; cash from operations is the money that’s really there. (Cash from operations over adjusted EBITDA.)

Every loan gets its own row.

Like keeping separate bills instead of one lumped total: each loan has its own line, its own due date, its own interest. You can see exactly when each one comes due and what it costs. (Per-instrument debt schedule with day-count interest.)

model_gate.py · In Plain Language

Every model runs a gate chain before it ships.

Each gate: what it looks for, why it matters. Blocking gates stop the ship; warnings flag for the human.

Until now this was three scripts a person ran by hand — and the summary could print “CLEAN — ship-eligible” on a model that was never even recalculated. Now it’s one gate that fails closed.
BLOCK
Was the math actually run? The numbers must be calculated by Excel itself, with proof it happened (a changed sentinel cell + a fresh save) — not a stale file.
BLOCK
Any broken cells? Zero spreadsheet errors (#REF!, #VALUE!, #DIV/0!) anywhere in the workbook.
BLOCK
Do the numbers tie to the filings? It re-adds revenue, balances the balance sheet, and cross-checks against the SEC filing. A mis-keyed tab can’t slip through.
BLOCK
Does each scenario move, the right way? Bull must beat Base; a dead toggle or an upside-down result is caught.
WARN
Where did every number come from? Color tells origin (blue = typed, green = linked, black = computed) and every input carries a source note.
In plain termsAn automatic proofreader for the spreadsheet. BLOCK stops the model from shipping; WARN just flags it for a human to glance at.

The Research Skill · How It Works

A ticker goes in. Three passes come out.

The spine of the skill is one architectural decision: ingestion and synthesis are two separate passes. Keeping them apart is what makes the view trustworthy.

PASS 1
INGEST
Primary-source-first. Every fact source-tier-flagged.
Walk sources in the primary-source-first order: 10-K business description first, then earnings transcripts and IR decks, then comparables, then third-party. Every fact carries a tier tag before synthesis begins. Tier 2 (vendor/sell-side) is supplementary and flagged. Tier 3 (web/opinion) is flagged loudest and never blended invisibly into a primary number.
T1: SEC filings T1: transcripts / IR decks T2: sell-side (flagged) T3: web (flagged loudest)
PASS 2
MODEL
Abbreviated three-statement on one tab. Cash over EBITDA, always.
P&L to cash-flow bridge to cap-stack to scenario toggles — single block. Adjusted EBITDA is the comps currency; every EBITDA figure bridges back to cash from operations and cash-less-capex. Skip the full balance sheet, full P&L detail, intermediate DCF years. One tab, one read.
CFO over adj. EBITDA cap-stack deep-dive (distressed) scenario toggles
PASS 3
SYNTH­ESIZE
The investment angle — run after ingestion completes, as a distinct pass.
Bull case, bear case, why-now, variant view (where the model breaks), and ranked key risks tied to the specific assumptions they would break. This pass reasons over the source-tiered base only — it never re-fetches and re-contaminates ingestion. This separation IS the product: the gap the method is designed to close.
bull / bear / why-now variant view key risks ranked

In plain terms Read everything first, mark exactly where each fact came from, then and only then form a view. A view built before ingestion finishes contaminates the research with what you wanted to find.

Controls · Source Tiers and Output Modes

Every fact carries a tier. Every run picks a mode.

Two orthogonal controls: the source-tier discipline determines how much to trust each number; the output mode selects how much to render.

Source-tier flagging
Tier 1 Primary — spine
  • SEC filings: 10-K, 10-Q, 8-K, DEF 14A, indentures
  • Company-issued transcripts and IR decks
  • EDGAR full-text search (free; zero connectors needed)
Tier 2 Secondary — supplementary, flagged
  • Sell-side / data-vendor (FactSet, S&P Global, Moody's, Daloopa)
  • Comps, consensus, ratings — useful but never blended silently into Tier 1
Tier 3 Web / opinion — flagged loudest
  • Last resort only; never substitutes a Tier-1 source
  • A number with no Tier-1 tag is suspect by default
Default connector preset: edgar-only — EDGAR plus company transcripts, zero vendor calls, every fact Tier 1. Switch to full-stack only when financial-data connectors are already authenticated; unauthenticated ones are logged as skipped, never silently replaced with web search. Bloomberg is not in the connector roster by design.
Output concision: --mode
full
All three artifacts rendered: the source-tiered ingestion background, the abbreviated one-tab model, and the full investment-angle synthesis with bull/bear/variant/risks. Default for a first-pass on a new name or a distressed credit. All three passes always run in full regardless of mode; mode controls only what is rendered.
exec-summary
Key Insights block first, 600 words maximum. Leads with the single most important insight, not a preamble. Abbreviated financials table, thesis, variant view, key risks, conclusion — all present but compressed. Built for a quick partner read or an IC pre-screen.
bullet
6–10 lines, approximately 200 words. Substantive key insight as line 1 — not a label, not a company description. No section headers. The format for a chat summary, a quick Slack drop, or a top-of-file elevator.

In plain terms Three passes always run. Mode only controls how much you see at the end — the full read, a compressed executive summary, or a six-line chat drop. Same analysis, different rendering.

WRONG: debt structure
Exec summary called it “a single $1.14B revolver.” The verifier pulled the Q1 2026 10-Q Note 7. The facility has two tranches under one New Credit Agreement: a $1.0B revolving commitment plus $450M in term loan commitments, both maturing Feb 27, 2029. $1.14B total outstanding is correct; calling it a single revolver understates structural complexity. WRONG: seller attribution
Exec summary called April 2026 unit sellers “sponsors.” The verifier pulled the 424B4 prospectus. The sellers were the IKAV and Sabinal acquisition counterparties (VEPU Inc., Simlog Inc., Sabinal Energy Operating). Tom Ward, the CEO, actually purchased 153K units in the same offering. The word “sponsors” conflated them with the PE backers (BCE/Bayou City). Fix applied.
Why this matters Both errors would have been invisible in any summary that blended Tier-1 and Tier-3 signals without flagging them. The two-pass architecture — ingest first, then synthesize, then verify synthesis against primary sources — is what surfaces the delta. The verifier is not a post-hoc check; it is part of the product.

What It Means

The human reviews judgment,
not arithmetic or provenance.

  • Every research run is source-tiered from the first fact — ingestion and synthesis are kept in separate passes so the view is never contaminated by what the analyst hoped to find.
  • Every model ships gated — an automatic proofreader re-runs the math, checks numbers against filings, and flips every assumption to confirm the model still behaves.
  • The verifier runs as an adversarial re-pull after synthesis — on MNR it caught two wrong facts (debt structure, seller attribution) before anything shipped.
  • The machine carries arithmetic, provenance, and tie-outs. That frees the analyst to review the one thing only a human can: the judgment.
  • The bar the method sets is met: checking it is no longer more work than doing it.
